## 什么是 龙虾超记忆防偷懒技能？

龙虾超记忆防偷懒技能 是一个**零依赖、纯 Markdown 的 AI 记忆框架**。与需要基础设施的向量数据库方案（Mem0、Chroma 等）不同，龙虾超记忆防偷懒技能 使用结构化的 Markdown 文件配合严格的执行协议来实现：

- ✅ **跨会话持久记忆** — AI 记得你是谁、你在做什么、你学到了什么
- ✅ **防偷懒强制执行** — 6 类偷懒行为识别 + 任务后六问法自检
- ✅ **用户审计权** — 你拥有所有记忆的最终解释权；AI 不能覆盖你的纠正
- ✅ **错误记忆隔离** — 错误的记忆被隔离，不会被反复强化
- ✅ **零配置** — 没有数据库、没有 API、没有依赖。只有 Markdown 文件。

### 为什么不用向量数据库？

| 方案 | 配置 | 规模 | 最适合 |
|------|------|------|--------|
| **龙虾超记忆防偷懒技能** (Markdown) | 零配置 | 10-1000 条 | 个人 AI 助手、编程代理、小团队 |
| Mem0 / Chroma | 需要基础设施 | 10K+ 条 | 企业聊天机器人、多用户系统 |
| LangChain Memory | 框架集成 | 中等 | 复杂代理流水线 |

**经验法则**：如果你的记忆能放进一个 Markdown 文件，你就不需要向量数据库。文本搜索足够了，而且你保留了完全的人类可读性。

## 🚀 快速开始（5 分钟）

### 第一步：创建记忆文件

```bash
# 复制模板
cp templates/SOUL.md ./SOUL.md
cp templates/MEMORY.md ./MEMORY.md
mkdir -p memory rules
```

### 第二步：配置 AI 身份（SOUL.md）

```markdown
# 代理身份

**名称**: 龙虾 🦞
**角色**: AI 编程搭档
**用户**: [你的名字]
**语言**: [你的语言]

## 核心规则
1. 每次会话前读取 MEMORY.md
2. 每次任务后运行防偷懒检查清单
3. 用户拥有所有记忆的最终解释权
```

### 第三步：开始使用

告诉你的 AI：*"遵循 龙虾超记忆防偷懒技能 协议。在回复前先读取 SOUL.md 和 MEMORY.md。"*

就这样。没有安装。没有配置文件。只有有纪律的 Markdown。

## 🦞 防偷懒框架（v3.2）

"龙虾"框架识别 6 类 AI 偷懒行为并强制纠正：

| 类型 | 表现 | 修复策略 |
|------|------|----------|
| **跳过型** | 工具/API 挂了不修，直接绕过去 | 先诊断再动手，失败 3 次换方案 |
| **轻信型** | 用户说什么信什么，不独立查证 | 交叉验证，官方文档 > 用户口述 |
| **表面型** | 做了但输出里不写关键内容 | 强制输出规范（见执行摘要） |
| **合并型** | 把多个步骤合并成一句带过 | 分步输出，每步独立可查 |
| **猜测型** | 不确定就瞎猜，不测试验证 | 测试替代猜测，写脚本验证假设 |
| **省略型** | 只写结果不写推导过程 | 必须展示完整推导链 |

**每次任务完成后，AI 必须回答 6 个问题：**

1. 步骤做完了没？ → 对照检查清单逐项勾选
2. 工具跑通了没？ → 检查是否有未修复的报错
3. 结果验证了没？ → 排除"应该没问题"心态
4. 输出了了没？ → 确认做了的事都说了
5. 记忆写了没？ → 确认当日日志已更新
6. 用户说法查证了没？ → 确认独立核实过

**全过才算合格，缺一就是偷懒。**

## 🛡️ 用户审计与错误隔离

### 自检悖论

所有 AI 铁律都是自我约束。如果 AI "忘了"检查清单、"判断"某条不重要、或"觉得"过期了该删——整套体系没有外部强制力。

**解决方案：用户拥有最终解释权。**

| 硬规则 | 描述 |
|--------|------|
| **审计权** | 用户有权随时查看、修改、删除 MEMORY.md 中的任何条目。AI 不得以"这是重要记忆"为由拒绝。 |
| **定期提醒** | 每周一启动时，如果距离上次用户审计超过 7 天，AI 必须提醒。 |
| **否决机制** | 用户指出记忆错误时，AI 必须立即标记 `[user-vetoed]`，后续不再引用。 |
| **审计日志** | 在 MEMORY.md 中维护审计记录。 |
