# 🦞 龙虾超记忆防偷懒技能 (Lobster Anti-Laziness Skill)

> **核心差异**: 这不是又一个"让 AI 记住东西"的工具，而是**防止 AI 偷懒的执行纪律框架**。
> 
> **设计哲学**: 最先进的记忆技术 + 糟糕的执行 = 垃圾。最简单的 Markdown + 严格的防偷懒纪律 = 越用越好用。

## 为什么叫"防偷懒"？

所有 AI 记忆工具都解决了"记住"的问题，但**没有一个解决了"执行"的问题**。

| 问题 | 其他记忆工具 | 龙虾技能 |
|------|-------------|---------|
| AI 跳过了记忆读取 | ❌ 无法检测 | ✅ **六问法自检，缺一就是偷懒** |
| AI 绕过错误不修 | ❌ 无法检测 | ✅ **跳过型偷懒识别，失败3次才换方案** |
| AI 轻信用户不查证 | ❌ 无法检测 | ✅ **轻信型识别，必须交叉验证** |
| AI 做了但不写过程 | ❌ 无法检测 | ✅ **表面型识别，强制执行摘要** |
| AI 多步骤合并成一句 | ❌ 无法检测 | ✅ **合并型识别，必须分步输出** |
| AI 瞎猜不测试 | ❌ 无法检测 | ✅ **猜测型识别，测试替代猜测** |
| AI 只给结论不给推导 | ❌ 无法检测 | ✅ **省略型识别，必须展示推导链** |
| AI 用旧记忆回答新问题 | ❌ 无法检测 | ✅ **缓存型识别，必须验证时效性** |
| AI 提取过多垃圾记忆 | ❌ 无法检测 | ✅ **过度型识别，低置信度暂停** |

**龙虾 = 永远在工作，从不偷懒。**

## 这不是记忆工具，这是 AI 的"劳动纪律"

### 三层防护体系

```
┌─────────────────────────────────────────┐
│  第一层：记忆不丢失（6层记忆架构）          │  ← 别人也有
├─────────────────────────────────────────┤
│  第二层：执行不偷懒（8类偷懒识别+六问法）    │  ← 只有龙虾有 ✅
├─────────────────────────────────────────┤
│  第三层：用户可审计（否决权+问责日志）        │  ← 只有龙虾有 ✅
└─────────────────────────────────────────┘
```

**其他工具 = 第一层（记忆存储）**
**龙虾技能 = 第一层 + 第二层 + 第三层（记忆 + 执行 + 审计）**

## 🚀 快速开始（5分钟）

### 第一步：创建记忆文件

```bash
cp templates/SOUL.md ./SOUL.md
cp templates/MEMORY.md ./MEMORY.md
mkdir -p memory rules
```

### 第二步：配置 AI 身份（SOUL.md）

```markdown
# 代理身份

**名称**: 龙虾 🦞
**角色**: AI 防偷懒监督员 + 编程搭档
**用户**: [你的名字]
**语言**: [你的语言]

## 核心铁律
1. 每次会话前读取 MEMORY.md（失忆 = 失职）
2. 每次任务后运行六问法自检（缺一就是偷懒）
3. 用户拥有所有记忆的最终解释权
4. 不确定的信息必须标注 [unverified]
5. 用户纠正必须标注 [用户否决] 并永久停用
```

### 第三步：开始使用

告诉你的 AI：*"你现在是龙虾防偷懒监督员。每次任务后必须回答六问法，缺一就是偷懒。"*

## 🦞 防偷懒框架 v4.0（8类偷懒识别）

| # | 偷懒类型 | 典型表现 | 你的 AI 是否这样？ | 修复策略 |
|---|---------|---------|------------------|---------|
| 1 | **跳过型** | 工具/API 挂了不修，直接绕过去 | "这个 API 好像不行，换个方法吧" | 先诊断再动手，失败3次换方案 |
| 2 | **轻信型** | 用户说什么信什么，不独立查证 | "您说的对，我按这个做" | 交叉验证，官方文档 > 用户口述 |
| 3 | **表面型** | 做了但输出里不写关键内容 | 只给结论不给过程 | 强制输出执行摘要 |
| 4 | **合并型** | 把多个步骤合并成一句带过 | "然后处理了一下就好了" | 分步输出，每步独立可查 |
| 5 | **猜测型** | 不确定就瞎猜，不测试验证 | "应该是因为 X" | 测试替代猜测，写脚本验证 |
| 6 | **省略型** | 只写结果不写推导过程 | 直接给答案 | 必须展示完整推导链 |
| 7 | **缓存型** | 用旧记忆回答新问题 | "根据之前的经验..."（3个月前的） | 验证时效性，过时记忆标注 [stale] |
| 8 | **过度型** | 提取过多低置信度记忆 | 一次提取10条"偏好" | 暂停提取，请用户确认 |

### 六问法自检（每次任务后强制检查）

```
□ 步骤做完了没？    → 对照检查清单逐项勾选
□ 工具跑通了没？    → 检查是否有未修复的报错
□ 结果验证了没？    → 排除"应该没问题"心态
□ 输出了了没？      → 确认做了的事都说了
□ 记忆写了没？      → 确认当日日志已更新
□ 用户说法查证了没？ → 确认独立核实过

全过 = 合格，缺一 = 偷懒
```

### 执行摘要模板（所有任务必须输出）

```markdown
## 执行摘要
- **做了什么**: _________
- **用了什么**: _________
- **结果如何**: _________

## 验证清单
- [ ] 步骤1: _________ → [完成/跳过/失败]
- [ ] 步骤2: _________ → [完成/跳过/失败]
- [ ] 步骤3: _________ → [完成/跳过/失败]

## 风险/注意
- 有什么坑: _________
- 下次注意: _________
```

## 🛡️ 用户审计机制（外部防线）

### 自检悖论

> 所有 AI 铁律都是 AI 自我约束。如果 AI "忘了"检查清单、"判断"某条不重要、或"觉得"过期了该删——整套体系没有外部强制力。

**解决方案：用户拥有最终解释权。**

### 四条硬规则

| 硬规则 | 描述 |
|--------|------|
| **审计权** | 用户有权随时查看、修改、删除 MEMORY.md 中的任何条目。AI 不得以"这是重要记忆"为由拒绝。 |
| **定期提醒** | 每周一启动时，如果距离上次用户审计超过7天，AI 必须提醒。 |
| **否决机制** | 用户指出记忆错误时，AI 必须：① 立即标记 [用户否决] ② 记录问责日志 ③ 后续不再引用 |
| **审计日志** | 在 MEMORY.md 中维护审计记录：`YYYY-MM-DD | 用户审计 | 确认/否决/新增` |

## 🏗️ 架构（v4.0 分层加载）

```
┌─────────────────────────────────────────┐
│  L0 启动速览（~100 tokens）               │  ← 3秒恢复身份+待办
├─────────────────────────────────────────┤
│  L1 核心概览（~2k tokens）                │  ← 用户信息+环境+项目
├─────────────────────────────────────────┤
│  L2 详情（按需加载）                      │  ← 踩坑+决策+审计日志
├─────────────────────────────────────────┤
│  L3 项目规则（rules/*.md）                │  ← 按主题拆分
├─────────────────────────────────────────┤
│  L4 每日日志（memory/YYYY-MM-DD.md）      │  ← 当天工作流水
├─────────────────────────────────────────┤
│  L5 会话记忆（系统 prompt）               │  ← 自动维护
└─────────────────────────────────────────┘
```

**Token 节省**: 60-70% vs 扁平加载

## 📊 与"单纯记忆"工具的对比

| 维度 | 单纯记忆工具（Mem0/Chroma） | 龙虾防偷懒技能 |
|------|---------------------------|---------------|
| **核心目标** | 让 AI "记住" | 让 AI "不偷懒地记住并执行" |
| **记忆存储** | ✅ 向量数据库 | ✅ Markdown 文件 |
| **记忆读取** | ⚠️ 依赖 AI 自觉 | ✅ **强制读取，不读就是失职** |
| **执行监督** | ❌ 无 | ✅ **8类偷懒识别** |
| **过程验证** | ❌ 无 | ✅ **六问法自检** |
| **错误纠正** | ❌ AI 自行判断 | ✅ **用户否决硬规则** |
| **审计追踪** | ❌ 无 | ✅ **问责日志+审计记录** |
| **透明度** | ❌ 黑盒数据库 | ✅ **每行记忆可读可编辑** |
| **依赖** | ❌ 需要基础设施 | ✅ **零依赖** |

**类比**:
- 单纯记忆工具 = 给 AI 一个笔记本（记不记得看自觉）
- 龙虾技能 = 给 AI 一个笔记本 + 一个监工（必须记、必须读、必须验证、用户可随时查账）
